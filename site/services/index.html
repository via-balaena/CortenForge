What we take on

Physics engine work

MuJoCo-aligned rigid-body dynamics in pure Rust: muscles, tendons, actuators, sensors, analytic derivatives, GPU batching. MJCF and URDF import. Hyperelastic soft-body FEM with contact and stress fields.

SDK extension

New physics, new materials, new fabrication targets, built as first-class components. Apache-2.0, so what we build for you doesn’t become a dependency on us.

RL & embodied-AI infrastructure

GPU-batched differentiable environments and a full learning stack: PPO, SAC, TD3, CEM.

Integration & architecture review

Build against the SDK on your problem, or have us assess the design before you commit to it.

How we build

The same standards the repository is held to apply to work we do for you.

Every crate passes an eight-criterion quality gate — coverage, documentation, lints, safety, dependency budget, layer integrity, WASM compatibility, and API surface — before anything lands on main. Correctness is validated against external ground truth where ground truth exists, and where it doesn’t, we say so rather than assert it.

Work delivered as SDK components stays yours under Apache-2.0. That is deliberate: the point of a kit is that you can keep building without us.
