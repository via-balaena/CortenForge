What we take on

Physics engine work

MuJoCo-aligned rigid-body dynamics in pure Rust: muscles, tendons, actuators, sensors, analytic and finite-difference derivatives, and a batched GPU pipeline. MJCF and URDF import. Hyperelastic soft-body FEM with contact and stress fields.

SDK extension

New physics, new materials, new fabrication targets, built as first-class components. Apache-2.0, so what we build for you doesn’t become a dependency on us.

RL & embodied-AI infrastructure

Parallel batched environments, a reverse-mode autograd tape, and a full learning stack: PPO, SAC, TD3, REINFORCE, CEM.

Integration & architecture review

Build against the SDK on your problem, or have us assess the design before you commit to it.

How we build

The same standards the repository is held to apply to work we do for you.

Every pull request grades the crates it touches and everything downstream of them — documentation, lints, unsafe-code review, a dependency budget, and layer integrity, with Layer-0 core crates additionally checked against the wasm32-unknown-unknown target. The full workspace is graded on main; coverage runs locally and in a weekly job; API design is reviewed by hand. Correctness is validated against external ground truth where ground truth exists, and where it doesn’t, we say so rather than assert it.

Components we build into the SDK ship under Apache-2.0, like the rest of it.
