A software development kit for the mechatronics and simulation space.

Composable Rust components for geometry, parametric design, meshing and fabrication, rigid- and soft-body physics, control, and reinforcement learning.

Get started View on GitHub

What it is

CortenForge provides the building blocks spanning the path physical → digital → physical. Because those components are general, they serve a wide range of applications — robotics, soft robotics, biomechanics, generative design and digital fabrication, custom-fit products, and embodied-AI research.

The kit is the product.

To prove the components compose end to end, our capstone undertaking — not yet built — is a differentiable body-to-device co-design loop for person-specific assistive robotics, with a powered, RL-controlled exoskeleton as the demonstration. See MISSION.md.

Get started

Applications depend on a single crate — the cortenforge facade — and reach the whole SDK through it, so the internal crate structure can evolve behind one stable contract.

[dependencies]
cortenforge = { git = "https://github.com/via-balaena/CortenForge" }

Depend on the repository, not crates.io. The published listings are years behind this workspace — cortenforge sits at 0.6.0 there and mesh-repair at 0.2.0, against 1.0.0 here — so cargo add will silently give you a stale version.

// Two umbrellas plus the design and fabrication crates, all through one dependency:
use cortenforge::sim;          // rigid + soft physics, soft↔rigid coupling, RL/opt
use cortenforge::mesh;         // load / repair / measure / print meshes
use cortenforge::cf_design;    // implicit-surface design kernel

The facade is a headless capability map across three domains — simulation and co-design (sim::core, sim::soft, the differentiable soft↔rigid coupling keystone sim::coupling, plus sim::rl, sim::opt, and sim::ml_chassis), mesh processing (mesh::io, mesh::repair, mesh::sdf, …), and design→fabrication, which is re-exported crate by crate rather than under an umbrella (cf_design, cf_scan_prep_core, cf_cast).

Bevy, GUI, and GPU crates are deliberately excluded from the facade, so anything compiled against it runs without a window or a graphics driver — on a build server, in a batch sweep, or inside a training loop. The GPU physics pipeline and the Bevy viewers are still in the repository; you depend on them directly when you want them.

git clone https://github.com/via-balaena/CortenForge.git
cd CortenForge
cargo build --workspace
cargo xtask grade <crate-name>   # run the quality gate on a single crate

The stack

Rigid-body physics

sim-core · sim-mjcf · sim-urdf

MuJoCo-aligned dynamics (79/79 conformance tests), MJCF and URDF import, analytic and finite-difference derivatives.

Soft-body physics

sim-soft

Hyperelastic FEM (Neo-Hookean / Yeoh), SDF→tet meshing, contact, differentiable.

ML, RL & optimization

sim-ml-chassis · sim-rl · sim-opt

VecEnv, autograd, CEM, REINFORCE, PPO, TD3, SAC, simulated annealing, parallel tempering.

Thermodynamic environments

sim-therm-env · sim-thermostat

Langevin thermostats, double wells, pairwise coupling, pluggable rewards.

Design

cf-design · cf-geometry · cf-spatial

SDF primitives, smooth booleans, mechanism assembly, MJCF and STL export.

Mesh

mesh-io · mesh-repair · mesh-sdf · +9 more

STL / OBJ / PLY / 3MF I/O, repair, offset, shell, lattice, print validation.

Scan → fabrication

cf-scan-prep-core · cf-cast · mesh-printability

Scan cleanup, multi-material mold generation, printability gating, procedure generation.

70 crates outside examples/. Pure Rust, with no framework dependencies in the physics and mesh cores. Layer-0 core crates are checked against the wasm32-unknown-unknown target as part of the quality gate.
