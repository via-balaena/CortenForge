# DT-97 Golden-File Flag Conformance Data

Golden `.npy` files generated by MuJoCo 3.4.0 for verifying CortenForge's runtime
flag behavior matches the reference implementation.

## Regenerating

```bash
uv pip install mujoco==3.4.0 numpy
uv run sim/L0/tests/scripts/gen_flag_golden.py
```

Requires MuJoCo **3.4.0 exactly** — the script aborts on version mismatch.
Running the script is idempotent (safe to re-run).

## Canonical Model

`flag_golden_test.xml` — enriched canonical model (nv=8, nq=9, nu=1):
- Two-link arm (hinge1 + hinge2) with spring, damper, frictionloss, tight limits
- Free-floating ball on floor (contacts at initial state)
- Fixed tendon coupling hinge1 and hinge2
- Equality constraint (hinge2 = 0.5 * hinge1)
- Joint position sensor on hinge1
- Motor actuator with ctrlrange [-1, 1]

All tests set `ctrl[0] = 2.0` (outside ctrlrange) so DISABLE_CLAMPCTRL has
observable effect on qacc.

Flags with "no" difference produce qacc identical to baseline. This is expected:
- DISABLE_SENSOR: sensors don't affect dynamics
- DISABLE_WARMSTART: solver converges to same solution
- DISABLE_MIDPHASE: broadphase produces same contacts for this model
- DISABLE_AUTORESET: no NaN/divergence in 10 steps
- DISABLE_NATIVECCD: no convex-convex collisions (sphere-plane only)
- DISABLE_ISLAND: same solver result regardless of island decomposition
- ENABLE_*: optional features that either don't affect qacc or aren't triggered

## Format

NumPy v1.0 `.npy` — little-endian float64, C-contiguous, shape `(10, 8)`.
10 steps, 8 DOFs (hinge1 + hinge2 + free body 6-DOF).
