## Problem Statement

CortenForge has 3,600+ tests but no systematic per-stage comparison against
MuJoCo reference data. Existing tests validate individual features in
isolation — they do not compare every pipeline stage output field-by-field
against MuJoCo's actual output. This is a **conformance gap**: without per-stage
reference comparison, numerical regressions at any pipeline stage can go
undetected if no feature-specific test happens to exercise the affected path.

MuJoCo computes derived quantities through a well-defined pipeline:
`mj_kinematics()` → `mj_crb()` → `mj_rne()` → `mj_passive()` →
`mj_collision()` → `mj_makeConstraint()` → `mj_projectConstraint()` →
`mj_fwdActuation()` → `mj_sensor*()` → `mj_tendon()`. Each stage writes
specific output fields to `mjData`. This spec creates a test for every
(model, stage) pair where reference data exists, comparing CortenForge's output
field-by-field against MuJoCo 3.4.0 reference `.npy` files.

### Key Behaviors

| Behavior | MuJoCo | CortenForge (current) |
|----------|--------|-----------------------|
| FK body poses from joint angles | `mj_kinematics()` computes xpos/xquat/xipos for all bodies via kinematic tree traversal | `data.forward(model)` → `mj_fwd_position()` → same tree traversal. Fields: `data.xpos: Vec<Vector3<f64>>`, `data.xquat: Vec<UnitQuaternion<f64>>`, `data.xipos: Vec<Vector3<f64>>` |
| Dense mass matrix | `mj_crb()` computes sparse `qM`, then `mj_fullM()` unpacks to dense nv×nv | `data.qM: DMatrix<f64>` stored dense. Direct comparison via `[(i,j)]` indexing |
| Coriolis + gravity bias | `mj_rne()` backward/forward Newton-Euler on kinematic tree | `data.qfrc_bias: DVector<f64>`. Same algorithm |
| Passive spring/damper forces | `mj_passive()` computes spring + damper forces per joint | `data.qfrc_passive: DVector<f64>`. Same algorithm |
| Contact detection | `mj_collision()` iterates broadphase geom pairs, runs narrowphase | `data.contacts: Vec<Contact>`. Geom pair order may differ — structural comparison needed |
| Contact depth convention | `d->contact[i].dist`: negative = penetration | `data.contacts[i].depth`: positive = penetration. **Sign flip required** |
| Contact normal | `d->contact[i].frame[0:3]` is contact normal | `data.contacts[i].normal: Vector3<f64>`. Direct match |
| Constraint Jacobian | `mj_makeConstraint()` assembles efc_J rows. Order: equality → friction loss → joint limits → tendon limits → contacts | `data.efc_J: DMatrix<f64>`. Same row ordering (verified in `assembly.rs:47-53`). Element-wise comparison valid when nefc matches |
| Constraint forces | `mj_projectConstraint()` iterative solver | `data.efc_force: DVector<f64>`. Looser tolerance (1e-4) for solver convergence |
| Actuator forces | `mj_fwdActuation()` applies gain/bias → transmission to joint space | `data.qfrc_actuator: DVector<f64>`, `data.actuator_force: Vec<f64>`. Direct computation |
| Sensor data | `mj_sensorPos/Vel/Acc()` writes to contiguous sensordata array | `data.sensordata: DVector<f64>`. Sensor order follows model definition |
| Tendon kinematics | `mj_tendon()` computes lengths and velocities | `data.ten_length: Vec<f64>`, `data.ten_velocity: Vec<f64>`. Direct computation |

### Convention Notes

| Field | MuJoCo Convention | CortenForge Convention | Porting Rule |
|-------|-------------------|------------------------|-------------|
| `xpos` | `d->xpos`: flat `mjtNum[nbody*3]` | `data.xpos: Vec<Vector3<f64>>` | Access: `ref_data[i*3+k]` ↔ `data.xpos[i][k]` |
| `xquat` | `d->xquat`: flat `[w,x,y,z]` per body | `data.xquat: Vec<UnitQuaternion<f64>>` | Access: `ref[i*4+0]` ↔ `q.w`, `ref[i*4+1]` ↔ `q.i`, `ref[i*4+2]` ↔ `q.j`, `ref[i*4+3]` ↔ `q.k`. Named accessors — no reordering needed. **Sign ambiguity:** q ≡ -q. Compare: `min(‖q_cf - q_ref‖, ‖q_cf + q_ref‖) < tol` |
| `xipos` | `d->xipos`: flat `mjtNum[nbody*3]` | `data.xipos: Vec<Vector3<f64>>` | Same as xpos |
| `qM` | `d->qM`: sparse (lower triangle). Dense via `mj_fullM()` | `data.qM: DMatrix<f64>` dense | Access: `ref[i*nv+j]` ↔ `data.qM[(i,j)]` |
| `qfrc_bias` | `d->qfrc_bias`: flat `mjtNum[nv]` | `data.qfrc_bias: DVector<f64>` | Access: `ref[i]` ↔ `data.qfrc_bias[i]` |
| `qfrc_passive` | `d->qfrc_passive`: flat `mjtNum[nv]` | `data.qfrc_passive: DVector<f64>` | Same as qfrc_bias |
| `qfrc_actuator` | `d->qfrc_actuator`: flat `mjtNum[nv]` | `data.qfrc_actuator: DVector<f64>` | Same as qfrc_bias |
| `actuator_force` | `d->actuator_force`: flat `mjtNum[nu]` | `data.actuator_force: Vec<f64>` | Access: `ref[i]` ↔ `data.actuator_force[i]` |
| `contact.dist` | Signed distance (negative = penetration) | `contact.depth` (positive = penetration) | **Negate:** `depth_cf ≈ -ref_dist` |
| `contact.frame` | `frame[0:3]` = contact normal | `contact.normal: Vector3<f64>` | Access: `ref_normal[k]` ↔ `contact.normal[k]` |
| `contact.geom` | `int[2]` geom pair | `contact.geom1: usize`, `contact.geom2: usize` | Cast: `ref_pair[0] as usize` ↔ `contact.geom1` |
| `sensordata` | `d->sensordata`: flat `mjtNum[nsensordata]` | `data.sensordata: DVector<f64>` | Access: `ref[i]` ↔ `data.sensordata[i]` |
| `ten_length` | `d->ten_length`: flat `mjtNum[ntendon]` | `data.ten_length: Vec<f64>` | Access: `ref[i]` ↔ `data.ten_length[i]` |
| `ten_velocity` | `d->ten_velocity`: flat `mjtNum[ntendon]` | `data.ten_velocity: Vec<f64>` | Same as ten_length |
| `efc_J` | `d->efc_J`: flat `mjtNum[nefc*nv]` row-major | `data.efc_J: DMatrix<f64>` | Access: `ref[i*nv+j]` ↔ `data.efc_J[(i,j)]` |
| `efc_b` | `d->efc_b`: flat `mjtNum[nefc]` | `data.efc_b: DVector<f64>` | Access: `ref[i]` ↔ `data.efc_b[i]` |
| `efc_force` | `d->efc_force`: flat `mjtNum[nefc]` | `data.efc_force: DVector<f64>` | Access: `ref[i]` ↔ `data.efc_force[i]` |

## Architecture Decisions

### AD-1: Single `layer_b.rs` file vs per-stage submodules

**Problem:** 9 pipeline stages × multiple models = 30–50 tests. Single file or submodule tree?

**Alternatives evaluated:**

| Option | Approach | Pros | Cons |
|--------|----------|------|------|
| 1 | Single `layer_b.rs` file | Simple, easy to grep, one module declaration | File may reach 1,500+ lines |
| 2 | Per-stage submodules under `layer_b/` | Clean separation, shorter files | More files, more mod declarations, shared helpers need re-export |

**Chosen:** Option 1 — single `layer_b.rs`. The file contains only test functions with
shared helpers at the top. Each test is self-contained (~20–30 lines). 50 tests at
~25 lines each = ~1,250 lines plus ~200 lines of helpers = ~1,450 lines total. This
is manageable and keeps the module structure flat. If the file grows beyond 2,000 lines
during implementation, split at that point.

### AD-2: Element-wise vs structural constraint comparison

**Problem:** Should constraint rows be matched structurally (by type + associated entity)
or compared element-wise?

**Alternatives evaluated:**

| Option | Approach | Pros | Cons |
|--------|----------|------|------|
| 1 | Element-wise (primary) with nefc pre-check | Simple. Valid because CortenForge uses same row ordering as MuJoCo (equality → friction → limits → contacts, verified in `assembly.rs:47-53`) | Fails silently if row ordering ever diverges |
| 2 | Structural matching by constraint type + entity | Robust to ordering changes | Complex, requires parsing constraint metadata |

**Chosen:** Option 1 — element-wise comparison with nefc pre-check. CortenForge's
constraint assembly matches MuJoCo's ordering (verified via debug_asserts at
`assembly.rs:796-826`). The nefc pre-check catches mismatches early. If nefc differs,
the test fails with a diagnostic message — it does not attempt structural fallback
(a nefc mismatch is a conformance gap, not a test infrastructure problem).

### AD-3: Quaternion comparison strategy

**Problem:** Quaternions have sign ambiguity (q ≡ -q represents the same rotation).
MuJoCo and CortenForge may produce quaternions with opposite signs.

**Chosen:** Compare using `min(‖q_cf - q_ref‖, ‖q_cf + q_ref‖) < tol` where the norm
is the L∞ norm over the 4 components. This handles sign ambiguity without requiring
sign canonicalization (which introduces branching and can mask real errors if applied
incorrectly).

## Specification

### S1. Common Infrastructure (Layer B helpers in `common.rs`)

**File:** `sim/L0/tests/mujoco_conformance/common.rs` (extend existing)
**MuJoCo equivalent:** N/A — test infrastructure
**Design decision:** Extend `common.rs` with Layer B-specific helpers. All helpers are
generic across stages — parameterized by model name, stage name, field name for
diagnostic output. The `parse_npy_i32()` function is added for contact geom pair loading.

**Additions:**

```rust
/// Parse a NumPy .npy v1.0 file with int32 (`<i4`) dtype.
/// Used for contact geom pair reference data.
#[allow(dead_code)]
pub fn parse_npy_i32(path: &std::path::Path) -> Option<(Vec<usize>, Vec<i32>)> {
    let bytes = std::fs::read(path).ok()?;
    if bytes.len() < 10 || &bytes[0..6] != b"\x93NUMPY" {
        panic!("invalid .npy magic");
    }
    let major = bytes[6];
    let minor = bytes[7];
    assert!(major == 1 && minor == 0, "only .npy v1.0 supported");
    let header_len = u16::from_le_bytes([bytes[8], bytes[9]]) as usize;
    let header_end = 10 + header_len;
    let header = std::str::from_utf8(&bytes[10..header_end]).expect("header not utf-8");
    assert!(
        header.contains("'<i4'") || header.contains("'int32'"),
        "expected int32 dtype, got: {header}"
    );
    assert!(
        header.contains("'fortran_order': False"),
        "only C-contiguous arrays supported"
    );
    let shape_start = header.find("'shape': (").expect("no shape in header") + 10;
    let shape_end = header[shape_start..].find(')').expect("no shape end") + shape_start;
    let shape_str = &header[shape_start..shape_end];
    let shape: Vec<usize> = shape_str
        .split(',')
        .filter(|s| !s.trim().is_empty())
        .map(|s| s.trim().parse().expect("bad shape dim"))
        .collect();
    let data_bytes = &bytes[header_end..];
    let n_elements: usize = shape.iter().product();
    assert_eq!(data_bytes.len(), n_elements * 4, "data size mismatch");
    let data: Vec<i32> = data_bytes
        .chunks_exact(4)
        .map(|chunk| i32::from_le_bytes(chunk.try_into().unwrap()))
        .collect();
    Some((shape, data))
}

/// Resolve path to a conformance reference data file.
#[allow(dead_code)]
pub fn reference_path(filename: &str) -> std::path::PathBuf {
    std::path::Path::new(env!("CARGO_MANIFEST_DIR"))
        .join("assets/golden/conformance/reference")
        .join(filename)
}

/// Resolve path to a conformance model file and load it.
#[allow(dead_code)]
pub fn load_conformance_model(name: &str) -> (sim_core::Model, sim_core::Data) {
    let path = std::path::Path::new(env!("CARGO_MANIFEST_DIR"))
        .join("assets/golden/conformance/models")
        .join(format!("{name}.xml"));
    let xml = std::fs::read_to_string(&path)
        .unwrap_or_else(|e| panic!("failed to read {}: {e}", path.display()));
    let model = sim_mjcf::load_model(&xml)
        .unwrap_or_else(|e| panic!("failed to load model {name}: {e}"));
    let data = model.make_data();
    (model, data)
}

/// Load reference .npy float64 data, panicking with a diagnostic message on failure.
#[allow(dead_code)]
pub fn load_reference_f64(model: &str, stage: &str, field: &str) -> (Vec<usize>, Vec<f64>) {
    let filename = format!("{model}_{stage}_{field}.npy");
    let path = reference_path(&filename);
    parse_npy(&path)
        .unwrap_or_else(|| panic!("failed to load reference: {}", path.display()))
}

/// Load reference .npy int32 data.
#[allow(dead_code)]
pub fn load_reference_i32(model: &str, stage: &str, field: &str) -> (Vec<usize>, Vec<i32>) {
    let filename = format!("{model}_{stage}_{field}.npy");
    let path = reference_path(&filename);
    parse_npy_i32(&path)
        .unwrap_or_else(|| panic!("failed to load reference: {}", path.display()))
}

/// Compare f64 arrays element-wise with diagnostic output on failure.
///
/// On mismatch, prints: `[{model}] {stage}.{field}[{index}]: expected {expected},
/// got {actual}, diff {diff}, tol {tol}`
///
/// Example failure output:
/// ```text
/// [pendulum] rne.qfrc_bias[0]: expected -1.471500000000000e+00,
///   got -1.471000000000000e+00, diff 5.000e-04, tol 1.000e-10
/// ```
#[allow(dead_code)]
pub fn assert_array_eq(
    model_name: &str,
    stage: &str,
    field: &str,
    expected: &[f64],
    actual: &[f64],
    tol: f64,
) {
    assert_eq!(
        expected.len(),
        actual.len(),
        "[{model_name}] {stage}.{field}: length mismatch: \
         expected {}, got {}",
        expected.len(),
        actual.len()
    );
    for (i, (e, a)) in expected.iter().zip(actual.iter()).enumerate() {
        let diff = (e - a).abs();
        assert!(
            diff <= tol,
            "[{model_name}] {stage}.{field}[{i}]: \
             expected {e:.15e}, got {a:.15e}, \
             diff {diff:.3e}, tol {tol:.3e}"
        );
    }
}

/// Compare quaternions with sign ambiguity handling (q ≡ -q).
///
/// Uses L∞ norm: `min(max|q_cf - q_ref|, max|q_cf + q_ref|) < tol`.
#[allow(dead_code)]
pub fn assert_quat_eq(
    model_name: &str,
    body_idx: usize,
    q_ref: [f64; 4], // [w, x, y, z] from MuJoCo reference
    q_cf: &nalgebra::UnitQuaternion<f64>,
    tol: f64,
) {
    let cf = [q_cf.w, q_cf.i, q_cf.j, q_cf.k];
    let dist_pos = (0..4).map(|k| (cf[k] - q_ref[k]).abs()).fold(0.0f64, f64::max);
    let dist_neg = (0..4).map(|k| (cf[k] + q_ref[k]).abs()).fold(0.0f64, f64::max);
    let dist = dist_pos.min(dist_neg);
    assert!(
        dist <= tol,
        "[{model_name}] fk.xquat[body {body_idx}]: \
         expected [{:.15e}, {:.15e}, {:.15e}, {:.15e}], \
         got [{:.15e}, {:.15e}, {:.15e}, {:.15e}], \
         min L∞ dist {dist:.3e}, tol {tol:.3e}",
        q_ref[0], q_ref[1], q_ref[2], q_ref[3],
        cf[0], cf[1], cf[2], cf[3]
    );
}
```

**AC coverage:** AC1 (infrastructure helpers exist and work).

### S2. FK Reference Tests

**File:** `sim/L0/tests/mujoco_conformance/layer_b.rs` (new)
**MuJoCo equivalent:** `mj_kinematics()` in `engine_core_smooth.c`
**Design decision:** Test all 8 models for FK. Each test loads the model, calls
`data.forward(&model)`, then compares `xpos`, `xquat`, and `xipos` against reference
data. Quaternion comparison uses `assert_quat_eq()` with sign ambiguity handling.
World body (body 0) is always tested — its xpos/xquat must be identity/origin.

**Test pattern (one test per model):**

```rust
#[test]
fn layer_b_fk_pendulum() {
    let (model, mut data) = common::load_conformance_model("pendulum");
    data.forward(&model).expect("forward");

    // xpos: nbody×3 — body positions in world frame
    let (shape, ref_xpos) = common::load_reference_f64("pendulum", "fk", "xpos");
    let nbody = shape[0];
    assert_eq!(nbody, data.xpos.len());
    let actual_xpos: Vec<f64> = data.xpos.iter().flat_map(|v| [v[0], v[1], v[2]]).collect();
    common::assert_array_eq("pendulum", "fk", "xpos", &ref_xpos, &actual_xpos, common::TOL_FK);

    // xquat: nbody×4 — body orientations (sign-ambiguous)
    let (_shape, ref_xquat) = common::load_reference_f64("pendulum", "fk", "xquat");
    for i in 0..nbody {
        let q_ref = [
            ref_xquat[i * 4],     // w
            ref_xquat[i * 4 + 1], // x
            ref_xquat[i * 4 + 2], // y
            ref_xquat[i * 4 + 3], // z
        ];
        common::assert_quat_eq("pendulum", i, q_ref, &data.xquat[i], common::TOL_FK);
    }

    // xipos: nbody×3 — inertial frame positions
    let (_shape, ref_xipos) = common::load_reference_f64("pendulum", "fk", "xipos");
    let actual_xipos: Vec<f64> = data.xipos.iter().flat_map(|v| [v[0], v[1], v[2]]).collect();
    common::assert_array_eq("pendulum", "fk", "xipos", &ref_xipos, &actual_xipos, common::TOL_FK);
}
```

**Models tested:** pendulum (nbody=2), double_pendulum (nbody=3),
contact_scenario (nbody=2), actuated_system (nbody=2), tendon_model (nbody=3),
sensor_model (nbody=3), equality_model (nbody=4), composite_model (nbody=5).

For actuated_system and composite_model, ctrl values must be set before
`forward()`: `data.ctrl[0] = 1.0` (and `data.ctrl[1] = 0.5` for actuated_system).
FK output does not depend on ctrl (FK is position-dependent only), but setting ctrl
ensures the full forward pass runs identically to the reference generation.

**AC coverage:** AC2–AC9 (one per model).

### S3. CRBA Reference Tests

**File:** `sim/L0/tests/mujoco_conformance/layer_b.rs`
**MuJoCo equivalent:** `mj_crb()` + `mj_fullM()` in `engine_core_smooth.c`

### S6. Collision Reference Tests

**File:** `sim/L0/tests/mujoco_conformance/layer_b.rs`
**MuJoCo equivalent:** `mj_collision()` in `engine_collision_driver.c`
**Design decision:** Structural comparison — contacts are matched by geom pair, not
array index. Algorithm: (1) load reference geom pairs via `parse_npy_i32()`,
(2) sort both reference and CortenForge contacts by `(geom1, geom2)`,
(3) assert same count, (4) compare depth (with sign flip: `depth_cf ≈ -ref_dist`),
normal, and position per matched pair. Tolerance: `TOL_COLLISION_DEPTH = 1e-6`.

**Test pattern:**

```rust
#[test]
fn layer_b_collision_contact_scenario() {
    let (model, mut data) = common::load_conformance_model("contact_scenario");
    data.forward(&model).expect("forward");

    // Load reference contact data
    let (_shape, ref_geom_pairs) = common::load_reference_i32(
        "contact_scenario", "contact", "geom_pairs",
    );
    let (_shape, ref_depth) = common::load_reference_f64(
        "contact_scenario", "contact", "depth",
    );
    let (_shape, ref_normal) = common::load_reference_f64(
        "contact_scenario", "contact", "normal",
    );
    let (_shape, ref_pos) = common::load_reference_f64(
        "contact_scenario", "contact", "pos",
    );

    let ref_ncon = ref_depth.len();

    // Build sorted reference contacts by geom pair
    let mut ref_contacts: Vec<(i32, i32, usize)> = (0..ref_ncon)
        .map(|i| (ref_geom_pairs[i * 2], ref_geom_pairs[i * 2 + 1], i))
        .collect();
    ref_contacts.sort_by_key(|&(g1, g2, _)| (g1, g2));

    // Build sorted CortenForge contacts by geom pair
    let ncon = data.ncon;
    let mut cf_contacts: Vec<(usize, usize, usize)> = (0..ncon)
        .map(|i| (data.contacts[i].geom1, data.contacts[i].geom2, i))
        .collect();
    cf_contacts.sort_by_key(|&(g1, g2, _)| (g1, g2));

    // Assert same contact count, listing geom pairs on mismatch
    if ref_ncon != ncon {
        let ref_pairs: Vec<_> = ref_contacts.iter().map(|(g1, g2, _)| (*g1, *g2)).collect();
        let cf_pairs: Vec<_> = cf_contacts.iter().map(|(g1, g2, _)| (*g1, *g2)).collect();
        panic!(
            "[contact_scenario] collision: expected {ref_ncon} contacts, got {ncon}.\n\
             MuJoCo geom pairs: {ref_pairs:?}\n\
             CortenForge geom pairs: {cf_pairs:?}"
        );
    }

    // Compare matched contacts
    for (ref_c, cf_c) in ref_contacts.iter().zip(cf_contacts.iter()) {
        let (ref_g1, ref_g2, ri) = *ref_c;
        let (cf_g1, cf_g2, ci) = *cf_c;
        assert_eq!(
            (ref_g1 as usize, ref_g2 as usize), (cf_g1, cf_g2),
            "[contact_scenario] collision: geom pair mismatch"
        );

        // Depth: sign flip (MuJoCo dist negative = penetration, CF depth positive)
        let ref_d = -ref_depth[ri]; // negate MuJoCo dist to get positive-penetration
        let cf_d = data.contacts[ci].depth;
        let diff = (ref_d - cf_d).abs();
        assert!(
            diff <= common::TOL_COLLISION_DEPTH,
            "[contact_scenario] collision.depth[geom pair ({cf_g1},{cf_g2})]: \
             expected {ref_d:.15e}, got {cf_d:.15e}, diff {diff:.3e}, \
             tol {:.3e}", common::TOL_COLLISION_DEPTH
        );

        // Normal
        for k in 0..3 {
            let ref_n = ref_normal[ri * 3 + k];
            let cf_n = data.contacts[ci].normal[k];
            let diff = (ref_n - cf_n).abs();
            assert!(
                diff <= common::TOL_COLLISION_DEPTH,
                "[contact_scenario] collision.normal[geom pair ({cf_g1},{cf_g2})][{k}]: \
                 expected {ref_n:.15e}, got {cf_n:.15e}, diff {diff:.3e}"
            );
        }

        // Position
        for k in 0..3 {
            let ref_p = ref_pos[ri * 3 + k];
            let cf_p = data.contacts[ci].pos[k];
            let diff = (ref_p - cf_p).abs();
            assert!(
                diff <= common::TOL_COLLISION_DEPTH,
                "[contact_scenario] collision.pos[geom pair ({cf_g1},{cf_g2})][{k}]: \
                 expected {ref_p:.15e}, got {cf_p:.15e}, diff {diff:.3e}"
            );
        }
    }
}
```

**Models tested:** contact_scenario (1 contact: sphere-plane), composite_model
(2 contacts: two balls on floor).

**AC coverage:** AC34–AC35 (one per model).

### S7. Constraint Reference Tests

**File:** `sim/L0/tests/mujoco_conformance/layer_b.rs`
**MuJoCo equivalent:** `mj_makeConstraint()` in `engine_core_constraint.c`,

**Design decision:** Element-wise comparison of efc_J, efc_b, efc_force. Per AD-2,

| Edge Case | Why it matters | Test(s) | AC(s) |
|-----------|---------------|---------|-------|
| World body (body 0) xpos/xquat | World body must have xpos=[0,0,0], xquat=[1,0,0,0] for all models. If wrong, entire FK chain is wrong. | T1–T8 (all FK tests compare body 0) | AC2–AC9 |

| Contact depth sign flip | MuJoCo `dist` is negative for penetration; CortenForge `depth` is positive. Forgetting to negate produces wrong comparison. | T33–T34 | AC34–AC35 |
| Models with ncon=0 | No collision test for models without contacts. Reference data only exists for contact_scenario and composite_model. | N/A — absence of test is intentional | N/A |
| Models with nu=0 | No actuation test for unactuated models. | N/A — absence is correct | N/A |
| Models with nsensor=0 | No sensor test for sensorless models. | N/A — absence is correct | N/A |
| Constraint row count (nefc) | If CortenForge produces different nefc than MuJoCo (e.g., different cone type), element-wise comparison is invalid. | T35–T37 (nefc pre-check) | AC36–AC38 |
| Pyramidal vs elliptic cone | Cone type affects constraint row count. Models must use MuJoCo default (pyramidal). | T35, T37 (contact models use default cone) | AC36, AC38 |
